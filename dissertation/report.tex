\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
A C++20 exchange infrastructure is presented for benchmarking five order book
data structure strategies across direct and gateway operating modes. The
framework contains a TCP order gateway, an incremental FIX parser, a lock-free
single-producer single-consumer queue, and a single-threaded matching engine
connected to interchangeable order book implementations. On the canonical
pinned-run dataset (2026-03-26), direct-indexed array access is the strongest
direct-mode implementation in six of seven scenarios, with mixed-scenario mean
latency of 210.72 ns and dense-full mean latency of 213.44 ns. In contrast,
sorted-vector insertion degrades sharply in dense books, reaching 16082.10 ns
in dense-full direct mode (75.3x slower than array). Gateway-mode latency is
dominated by network and transport overhead for non-vector rows (median network
share 98.7\%), which reduces the visible contribution of internal order-book
differences to end-to-end timing. The evidence supports two bounded
conclusions. First, data-structure execution path and memory-access behavior
are the dominant factors when algorithmic work is isolated. Second, in
end-to-end gateway operation, system-level bottlenecks can mask data-structure
gains unless transport and parsing overhead are reduced. The project
contributes a reproducible benchmark harness, scenario-driven workload
generation, and decomposed network/queue/engine timing suitable for future experiments
on kernel-bypass networking and multi-producer ingestion.
